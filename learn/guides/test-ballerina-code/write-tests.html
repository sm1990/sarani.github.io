      Define a test

      The test module provides the necessary annotations to construct a test suite. Therefore, importing the test module is essential in order to write Ballerina tests.

      Once the test module is imported, the following annotation can be used to write a test function.

      The function defined after the above annotation will be detected as a test function. This annotation supports the following attributes in order to configure the test execution.

      • enable: {true|false} - Enable/disable the test. The default value is true.

      • before: <function name> - The function to be run just before the test is run. The default value is nil.

      • after: <function name> - The function to be run just after the test is run. The default value is nil.

      • dependsOn: [<function names>, …] - List of functions on which the test function depends. The order in which the comma-separated list appears has no prominence. In case there needs to be an order, the dependsOn parameter can be used to create an ordered sequence of functions with one function depending on the other.

      • dataProvider: <function name> - Specifies the function, which will be used to provide the data sets for the test.

      • groups: [“<test group name”, …] - A comma-separated list of test group names (one or more) to which this test belongs.

      Use assertions

      The Ballerina test framework supports the following assertions, which help to verify the expected behavior of a piece of code. These assertions can be used to decide if the test is passing or failing based on the condition.

      Assertion functionDescription
      assertTrue(boolean expression, string message) Asserts that the expression is true with an optional message.
      assertFalse(boolean expression, string message) Asserts that the expression is false with an optional message.
      assertEquals(anydata|error actual, anydata expected, string message) Asserts that the actual value is equal to the expected value with an optional message.
      assertNotEquals(anydata actual, anydata expected, string message) Asserts that the actual value is not equal to the expected value with an optional message.
      assertExactEquals(any|error actual, any|error expected, string message) Asserts that the actual entity is exactly equal to the expected entity with an optional message.
      assertNotExactEquals(any|error actual, any|error expected, string message) Asserts that the actual entity is not exactly equal to the expected entity with an optional message.
      assertFail(string message) Fails the test. This is useful to fail a test based on a check for a condition while it is in execution.
